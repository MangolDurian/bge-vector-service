# 部署文档

> BGE Vector Service 服务器部署指南
> 目标环境：Linux / Python 3.9+ / 10.1.38.245:7999

---

## 1. 服务器要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | CentOS 7+ / Ubuntu 18.04+ | Ubuntu 20.04+ |
| Python | 3.9+ | 3.10 或 3.11 |
| CPU | 4 核 | 8 核+ |
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 5 GB（模型缓存） | 10 GB+ |
| GPU | 不需要 | NVIDIA GPU + CUDA 11.7+（可选，加速推理） |

---

## 2. 部署步骤

### 2.1 上传项目

```bash
# 方式一：git clone（服务器需要能访问 GitHub）
cd /opt
git clone https://github.com/MangolDurian/bge-vector-service.git
cd bge-vector-service

# 方式二：本地打包上传
# 在本地执行：
tar -czf bge-vector-service.tar.gz -C /path/to/bge-vector-service .
scp bge-vector-service.tar.gz user@10.1.38.245:/opt/

# 在服务器执行：
cd /opt
mkdir -p bge-vector-service && cd bge-vector-service
tar -xzf ../bge-vector-service.tar.gz
```

### 2.2 安装依赖

```bash
cd /opt/bge-vector-service

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

> 如果服务器无法访问 PyPI，配置国内镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 2.3 首次启动（下载模型）

首次启动会自动从 HuggingFace 下载模型文件（约 1.2GB）：

```bash
# 设置模型为 bge-large-zh-v1.5（1024 维）
export EMBEDDING_MODEL_NAME=BAAI/bge-large-zh-v1.5

# 如果服务器无法访问 HuggingFace，设置镜像
export HF_ENDPOINT=https://hf-mirror.com

# 启动服务
python main.py
```

等待看到以下日志即表示启动成功：

```
[启动] 正在加载模型: BAAI/bge-large-zh-v1.5 ...
Loading weights: 100%|██████████| 391/391
[启动] 模型加载完成
INFO:     Uvicorn running on http://0.0.0.0:7999
```

按 `Ctrl+C` 停止服务，模型已缓存到本地，后续启动无需重新下载。

### 2.4 验证服务

```bash
# 健康检查
curl http://10.1.38.245:7999/health

# 预期返回：
# {"status":"ok","model":"BAAI/bge-large-zh-v1.5"}

# 测试向量生成
curl -X POST http://10.1.38.245:7999/vector \
  -H "Content-Type: application/json" \
  -d '{"keyword": "测试文本"}'

# 预期返回：
# {"vector":[0.0123,-0.0345,...]}
```

## 3. 后台运行（生产部署）

### 方式一：nohup（简单）

```bash
cd /opt/bge-vector-service
source venv/bin/activate

export EMBEDDING_MODEL_NAME=BAAI/bge-large-zh-v1.5

nohup python main.py > logs/service.log 2>&1 &
```

日志文件位于 `logs/service.log`。

### 方式二：systemd 服务（推荐）

创建服务文件：

```bash
sudo vi /etc/systemd/system/bge-vector.service
```

写入以下内容（根据实际路径修改）：

```ini
[Unit]
Description=BGE Vector Service - 中文文本向量服务
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/bge-vector-service
Environment=EMBEDDING_MODEL_NAME=BAAI/bge-large-zh-v1.5
ExecStart=/opt/bge-vector-service/venv/bin/python main.py
Restart=always
RestartSec=10
StandardOutput=append:/opt/bge-vector-service/logs/service.log
StandardError=append:/opt/bge-vector-service/logs/error.log

[Install]
WantedBy=multi-user.target
```

启动与管理：

```bash
# 创建日志目录
mkdir -p /opt/bge-vector-service/logs

# 启动服务
sudo systemctl daemon-reload
sudo systemctl start bge-vector

# 设置开机自启
sudo systemctl enable bge-vector

# 查看状态
sudo systemctl status bge-vector

# 停止服务
sudo systemctl stop bge-vector

# 重启服务
sudo systemctl restart bge-vector

# 查看日志
tail -f /opt/bge-vector-service/logs/service.log
```

## 4. Java 端对接配置

服务部署完成后，Java 项目中修改配置：

```yaml
custom:
  q: "http://10.1.38.245:7999/vector"
  a: "http://10.1.38.245:7999/answer"
```

对接关系：

| Java 方法 | 调用接口 | 用途 |
|-----------|---------|------|
| `NlpService.calculateVector()` | `POST /vector` | 字段值入库时生成向量 |
| `NlpService.word2Vector()` | `POST /vector` | 字段值向量化 |
| 自然语言查询 | `POST /answer` | 检索时查询向量化 |

请求格式（两端一致）：

```json
{"keyword": "北京市2025年吉林1号0.5米分辨率全色遥感影像"}
```

响应格式（两端一致）：

```json
{"vector": [0.0123, -0.0345, ...]}
```

## 5. 前端 UI（可选）

`web/index.html` 是辅助测试页面，运维部署时可部署也可不部署。若希望外部直接通过 `ip + 端口` 进入前端，可在服务器上使用独立端口提供静态页面：

```bash
cd /opt/bge-vector-service
python -m http.server 8001 -d web
```

访问地址：

```text
http://10.1.38.245:8001
```

前端页面只负责辅助调用接口，后端 API 仍由 `7999` 端口提供。浏览器访问 `http://10.1.38.245:8001` 时，页面会请求同一主机的 `http://10.1.38.245:7999`。

如果需要长期运行，可单独配置 systemd 服务：

```ini
[Unit]
Description=BGE Vector Service UI
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/bge-vector-service
ExecStart=/usr/bin/python3 -m http.server 8001 -d web
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

## 6. 环境变量配置

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `EMBEDDING_MODEL_NAME` | `BAAI/bge-base-zh-v1.5` | 模型名称，**部署时需改为 `BAAI/bge-large-zh-v1.5`** |
| `EMBEDDING_USE_FP16` | `true` | 半精度推理，GPU 下加速 |
| `EMBEDDING_DEVICE` | 自动 | 推理设备（`cpu` / `cuda`） |
| `EMBEDDING_HOST` | `0.0.0.0` | 监听地址 |
| `EMBEDDING_PORT` | `7999` | 监听端口 |
| `EMBEDDING_BATCH_SIZE` | `32` | 批量编码大小 |
| `EMBEDDING_MAX_TEXTS` | `128` | 单次最大文本数 |
| `HF_ENDPOINT` | 默认 HuggingFace | 模型下载镜像（国内推荐 `https://hf-mirror.com`） |

## 8. 常见问题

### 模型下载慢或失败

```bash
export HF_ENDPOINT=https://hf-mirror.com
python main.py
```

### 端口被占用

```bash
# 查看占用进程
lsof -i :7999

# 释放端口
kill -9 <PID>

# 或修改端口
export EMBEDDING_PORT=9000
python main.py
```

### 内存不足

bge-large-zh-v1.5 运行时约占用 2~3GB 内存。如果服务器内存紧张，可切换到更小的模型：

```bash
export EMBEDDING_MODEL_NAME=BAAI/bge-base-zh-v1.5   # 768维，约400MB
# 或
export EMBEDDING_MODEL_NAME=BAAI/bge-small-zh-v1.5   # 512维，约100MB
```

### GPU 加速

如果服务器有 NVIDIA GPU 并已安装 CUDA：

```bash
export EMBEDDING_DEVICE=cuda
python main.py
```

### 防火墙放行

```bash
# CentOS
firewall-cmd --permanent --add-port=7999/tcp
firewall-cmd --permanent --add-port=8001/tcp  # 如部署前端 UI
firewall-cmd --reload

# Ubuntu
ufw allow 7999/tcp
ufw allow 8001/tcp  # 如部署前端 UI
```
