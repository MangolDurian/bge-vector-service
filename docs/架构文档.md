# 架构文档

> 中文文本向量服务的系统架构与技术选型说明

---

## 1. 项目结构

```
embedding-api/
├── main.py              # FastAPI 应用入口，定义路由与接口
├── config.py            # 集中配置管理，支持环境变量覆盖
├── requirements.txt     # Python 依赖声明
└── docs/
    ├── API接口文档.md
    ├── 使用文档.md
    ├── 架构文档.md       ← 本文件
    └── 升级指南.md
```

---

## 2. 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 框架 | FastAPI | 高性能异步框架，自动生成 OpenAPI 文档 |
| ASGI 服务器 | Uvicorn | 基于 uvloop 的高性能 ASGI 实现 |
| 向量模型 | BGE (FlagEmbedding) | 智源研究院开源的中文向量模型 |
| 模型名称 | bge-base-zh-v1.5 | 768 维，中文专用，体积与效果平衡 |
| 数据校验 | Pydantic v2 | 请求/响应自动校验与序列化 |

---

## 3. 请求处理流程

```
客户端请求
    │
    ▼
┌──────────────┐
│   Uvicorn    │  ASGI 服务器，接收 HTTP 请求
└──────┬───────┘
       │
       ▼
┌──────────────┐
│   FastAPI    │  路由匹配 + Pydantic 参数校验
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  FlagEmbedding│  模型推理，文本 → 向量
│  (bge-base)  │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  JSON 响应   │  序列化向量结果返回给客户端
└──────────────┘
```

### 生命周期

1. **启动阶段**（`lifespan`）：加载模型到内存/显存，约需 5~15 秒
2. **运行阶段**：接收请求，调用模型编码，返回向量
3. **关闭阶段**：释放模型资源

## 4. 配置管理

所有配置集中在 `config.py`，通过 `Settings` 类统一管理：

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|---------|--------|------|
| MODEL_NAME | EMBEDDING_MODEL_NAME | BAAI/bge-base-zh-v1.5 | 模型名称 |
| QUERY_INSTRUCTION | EMBEDDING_QUERY_INSTRUCTION | 为这个句子生成表示... | 检索指令前缀 |
| USE_FP16 | EMBEDDING_USE_FP16 | true | 半精度推理 |
| DEVICE | EMBEDDING_DEVICE | 自动 | 推理设备 |
| BATCH_SIZE | EMBEDDING_BATCH_SIZE | 32 | 编码批大小 |
| HOST | EMBEDDING_HOST | 0.0.0.0 | 监听地址 |
| PORT | EMBEDDING_PORT | 7999 | 后端 API 监听端口 |
| MAX_TEXTS | EMBEDDING_MAX_TEXTS | 128 | 单次最大文本数 |

优先级：环境变量 > 代码默认值。

## 6. 扩展点

当前架构为单服务单模型，以下是可扩展的方向（详见升级指南）：

- **前端 UI**：`web/index.html` 为可选辅助测试页面，可通过 `8001` 端口独立提供静态访问
- **模型热切换**：通过配置切换不同 BGE 模型
- **多模型并行**：支持同时加载多个模型，按需调用
- **批处理优化**：引入请求队列，合并短时间内的多个请求
- **缓存层**：对相同文本的向量结果进行缓存
- **Docker 部署**：容器化部署，方便运维
