# 升级指南

> 服务后续可持续升级的方向与具体步骤

## 1. 切换模型

修改环境变量即可切换，无需改代码：

```bash
# 切换到更精准的大模型（向量维度 1024）
export EMBEDDING_MODEL_NAME=BAAI/bge-large-zh-v1.5
python main.py

# 切换到多语言模型（支持 100+ 语言，最大 8192 tokens）
export EMBEDDING_MODEL_NAME=BAAI/bge-m3
python main.py

# 切换到轻量模型（更快，向量维度 512）
export EMBEDDING_MODEL_NAME=BAAI/bge-small-zh-v1.5
python main.py
```

| 模型 | 维度 | 最大长度 | 大小 | 适用场景 |
|------|------|---------|------|---------|
| bge-small-zh-v1.5 | 512 | 512 | ~100MB | 轻量快速 |
| bge-base-zh-v1.5 | 768 | 512 | ~400MB | 平衡之选（当前） |
| bge-large-zh-v1.5 | 1024 | 512 | ~1.2GB | 最高精度 |
| bge-m3 | 1024 | 8192 | ~1.2GB | 多语言 + 长文本 |

## 2. Docker 容器化部署

创建 `Dockerfile`：

```dockerfile
FROM python:3.11-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000
CMD ["python", "main.py"]
```

构建与运行：

```bash
docker build -t embedding-api .
docker run -d -p 8000:8000 embedding-api
```

GPU 推理需要使用 NVIDIA Container Toolkit 并添加 `--gpus all`。

## 3. 添加认证

在 `main.py` 中添加 API Key 认证中间件：

```python
from fastapi import Depends, HTTPException, Security
from fastapi.security import APIKeyHeader

API_KEY = "your-secret-key"
api_key_header = APIKeyHeader(name="X-API-Key")

async def verify_api_key(key: str = Security(api_key_header)):
    if key != API_KEY:
        raise HTTPException(status_code=401, detail="Invalid API Key")
    return key
```

## 4. 添加重排序接口（Reranker）

BGE 系列提供 Reranker 模型，可新增接口提升检索精度：

```python
from FlagEmbedding import FlagReranker

reranker = FlagReranker("BAAI/bge-reranker-base", use_fp16=True)

@app.post("/rerank")
def rerank(query: str, documents: list[str]):
    pairs = [[query, doc] for doc in documents]
    scores = reranker.compute_score(pairs)
    return {"scores": scores}
```

## 7. 监控与日志

添加结构化日志和 Prometheus 指标：

```bash
pip install prometheus-fastapi-instrumentator
```

```python
from prometheus_fastapi_instrumentator import Instrumentator

Instrumentator().instrument(app).expose(app)
```

访问 `http://localhost:8000/metrics` 获取指标。

## 8. 升级路径建议

按需逐步升级，无需一步到位：

| 阶段 | 升级内容 | 优先级 |
|------|---------|--------|
| 第一阶段 | Docker 部署 + 环境变量配置 | 高 |
| 第二阶段 | API Key 认证 | 高 |
| 第三阶段 | 结构化日志 + 健康监控 | 中 |
| 第四阶段 | Reranker 接口 | 按需 |
| 第五阶段 | 批处理队列 + 缓存 | 高并发时 |
| 第六阶段 | 多模型并行 | 特定场景 |
